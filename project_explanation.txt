================================================================================
                    LPU CONNECT - PROJECT EXPLANATION
================================================================================

PROJECT NAME: LPU Connect
VERSION: 0.1.0
DESCRIPTION: A comprehensive campus community platform for LPU (Lovely Professional 
University) students to connect, share, buy/sell items, discover events, and build 
meaningful relationships within the campus ecosystem.

================================================================================
                        PROJECT OVERVIEW
================================================================================

LPU Connect is a full-stack web application built with modern technologies that 
serves as an all-in-one platform for campus life. The platform enables students to:

1. Discover and attend campus events
2. Buy and sell surplus items (books, gadgets, furniture, etc.)
3. Engage in community discussions and forums
4. Create and manage student profiles
5. Navigate campus hotspots and rate locations
6. Connect with peers and build meaningful friendships

================================================================================
                      TECHNOLOGY STACK
================================================================================

FRONTEND:
- Framework: Next.js 16.1.1 (React 19.2.3) with App Router
- Styling: Tailwind CSS 3.4.19 with CSS variables
- UI Components: Lucide React icons (0.562.0)
- Animations: Framer Motion 12.23.26
- Authentication: NextAuth 4.24.13 with JWT strategy
- Language: TypeScript 5

BACKEND:
- Runtime: Node.js with Next.js API routes
- Database: MongoDB 5.x with Mongoose ODM 9.1.1
- Authentication: NextAuth with Credentials provider and JWT tokens
- Security: bcryptjs 3.0.3 for password hashing

BUILD & TOOLING:
- Build Tool: Turbopack (integrated with Next.js 16.1.1)
- Linting: ESLint 9
- CSS Processing: PostCSS 8.5.6, Autoprefixer 10.4.23
- Utility Libraries: clsx 2.1.1, tailwind-merge 3.4.0

================================================================================
                      DATA MODELS
================================================================================

USER MODEL (User.ts)
{
  _id: ObjectId (Primary Key)
  name: String (Required)
  email: String (Required, Unique)
  password: String (Required, Hashed)
  image: String (Profile photo URL)
  avatar: String (Emoji avatar ID, default: 'boy1')
  role: String (Enum: 'student'|'admin'|'moderator', default: 'student')
  bio: String (Max 200 chars)
  courses: [String] (Array of course names)
  interests: [String] (Array of interest tags)
  socialLinks: {
    linkedin: String,
    github: String,
    twitter: String,
    instagram: String
  }
  createdAt: Date (Auto-generated)
}

EVENT MODEL (Event.ts)
{
  _id: ObjectId (Primary Key)
  title: String (Required)
  description: String
  date: Date (Required)
  time: String (Required, HH:MM format)
  location: String (Required)
  image: String (Event image URL)
  organizerId: ObjectId (Ref: User, Required)
  attendees: [ObjectId] (Ref: User, Array of attendee IDs)
  createdAt: Date (Auto-generated)
}

PRODUCT MODEL (Product.ts)
{
  _id: ObjectId (Primary Key)
  name: String (Required)
  price: Number (Required)
  description: String
  category: String (Enum: Textbooks|Electronics|Furniture|Clothing|Other)
  image: String (Product image URL)
  sellerId: ObjectId (Ref: User, Required)
  status: String (Enum: available|in-bid|sold, default: available)
  bids: [{
    userId: ObjectId (Ref: User),
    amount: Number,
    message: String,
    createdAt: Date
  }]
  createdAt: Date (Auto-generated)
}

POST MODEL (Post.ts)
{
  _id: ObjectId (Primary Key)
  content: String (Required)
  category: String (Enum: General|Clubs|Academic|Events|Other)
  authorId: ObjectId (Ref: User, Required)
  likes: [ObjectId] (Array of user IDs who liked)
  comments: [{
    userId: ObjectId (Ref: User),
    text: String,
    createdAt: Date
  }]
  createdAt: Date (Auto-generated)
}

LOCATION MODEL (Location.ts)
{
  _id: ObjectId (Primary Key)
  name: String (Required)
  type: String (hostel|cafe|library|sports|other)
  description: String
  latitude: Number
  longitude: Number
  ratings: [Number] (Array of ratings 1-5)
  reviews: [{
    userId: ObjectId (Ref: User),
    text: String,
    rating: Number,
    createdAt: Date
  }]
  createdAt: Date (Auto-generated)
}

================================================================================
                      AUTHENTICATION FLOW
================================================================================

1. USER REGISTRATION:
   - User fills registration form with name, email, password
   - Password is hashed using bcryptjs (12 salt rounds)
   - User record created in MongoDB
   - User redirected to login page

2. USER LOGIN:
   - User enters email and password
   - Password compared against stored hash
   - JWT token generated with user data (id, email, name, role, avatar)
   - Session stored in NextAuth callback
   - User redirected to home page

3. SESSION MANAGEMENT:
   - JWT token expires after session duration
   - Token payload includes: id, email, name, role, avatar
   - Session data available via useSession() hook
   - Session update triggers JWT refresh with trigger='update'
   - localStorage used for theme persistence

4. ROLE-BASED ACCESS:
   - student: Default role for new users
   - admin: Can delete any content, view all attendees/bidders
   - moderator: Similar to admin
